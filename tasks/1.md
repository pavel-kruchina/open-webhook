# Task 1 — Rebrand "WebHook Tester" → "open-webhook" (proper fork)

## Goal
Rebrand the application from "WebHook Tester" to "open-webhook" while keeping clear attribution that it is
a fork of the upstream project.

## Acceptance criteria

1. **Frontend branding**
   - Replace the "webhook tester" branding shown to users with "open-webhook".
   - The logo/image in the top-left corner: replace the image with text (e.g. "open-webhook").
   - The repository link near "Help" must point to `https://github.com/pavel-kruchina/open-webhook`.
   - Update the browser tab title / page `<title>` and any visible product name.

2. **Docs / info files**
   - Update `README.md` and other info files (AGENTS.md, Helm chart docs, etc.) to use the new name.
   - Keep a clear "forked from" reference/attribution to the upstream `tarampampam/webhook-tester`, so it reads
     as a proper fork (do not erase upstream credit).
   - Remove the AI-related sections from the README ("A note on AI-assisted development" and
     "AI Agent Instructions").
   - Verify the README for features we no longer support (e.g. memory/fs storage drivers) and correct them.
   - Add documentation about the new multipart file storage + download feature (already partially present).

## Out of scope
- No behavioral/code-logic changes beyond branding strings, the logo→text swap, and the repo link.
- Backend Go module path (`gh.tarampamp.am/webhook-tester/v2`) stays as-is (changing it is a large, risky
  refactor and unrelated to user-facing branding).
